## Story
**As a** gestor do Mr. Lion,
**I want** receber notificações push no celular quando um produto atingir o nível crítico de estoque,
**so that** eu seja alertado proativamente mesmo quando não estou com o app aberto.

## Acceptance Criteria

1. Na primeira visita após login, o app solicita permissão de notificações push ao usuário
2. O usuário pode aceitar ou recusar a permissão sem impacto funcional no app
3. Quando qualquer produto tem `current_stock <= min_stock` após uma movimentação, uma notificação push é enviada
4. A notificação push exibe: título "⚠️ Estoque Crítico — Mr. Lion", corpo com nome e quantidade atual do produto
5. Múltiplos produtos críticos na mesma operação geram uma única notificação agrupada: "3 produtos em nível crítico"
6. Ao clicar na notificação, o app abre na página de Produtos com filtro "Critical" ativo
7. As notificações funcionam quando o app está em background (service worker)
8. Em modo mock, as notificações são simuladas via toast (sem SW real)
9. Uma configuração em Settings (ou no header) permite o usuário desativar notificações push
10. A funcionalidade usa apenas a Web Push API nativa (sem Firebase/serviço terceiro)

### API Web Push Nativa
```typescript
// pushNotifications.ts
export function isSupported(): boolean {
  return 'Notification' in window
}

export async function requestPermission(): Promise<NotificationPermission> {
  if (!isSupported()) return 'denied'
  return Notification.requestPermission()
}

export async function sendLocalNotification(title: string, options: NotificationOptions) {
  if (Notification.permission !== 'granted') return
  if (!localStorage.getItem('push_notifications_enabled')) return

  const registration = await navigator.serviceWorker.ready
  registration.showNotification(title, {
    icon: '/icon-192x192.png',
    badge: '/icon-192x192.png',
    ...options
  })
}
```

### Nota importante sobre SW e vite-plugin-pwa
O projeto usa `vite-plugin-pwa` que gera SW automaticamente. Para adicionar handler `notificationclick`, usar a estratégia `injectManifest` ou criar um SW custom em `public/sw-custom.js`. Verificar configuração atual em `vite.config.ts`.
