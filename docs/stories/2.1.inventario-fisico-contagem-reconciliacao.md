## Story
**As a** gestor do Mr. Lion,
**I want** realizar uma contagem física do estoque e reconciliar automaticamente as divergências,
**so that** o sistema reflita sempre a realidade física do estoque e eu identifique perdas e erros.

## Acceptance Criteria

1. Existe uma nova seção "Inventário" acessível via menu (ícone ClipboardCheck) com rota `/inventario`
2. A tela exibe todos os produtos ativos com campo para digitar a quantidade contada fisicamente
3. O sistema calcula e exibe a divergência: `contado - sistema = divergência` (positivo em verde, negativo em vermelho)
4. O usuário pode navegar pelos campos com Tab (acessibilidade de teclado)
5. Botão "Iniciar Inventário" cria um snapshot do estoque atual antes de começar
6. Botão "Finalizar Inventário" cria movimentações de `adjustment` para TODOS os produtos com divergência ≠ 0
7. Cada ajuste automático tem nota: `"Ajuste de inventário - Contagem: {contado}, Sistema: {sistema}"`
8. Um relatório de inventário é exibido pós-finalização: produtos sem divergência, com divergência positiva, com divergência negativa
9. É possível cancelar o inventário sem salvar alterações
10. Apenas managers podem iniciar/finalizar inventário; operators podem apenas visualizar resultados

### Lógica de ajuste
```typescript
for (const product of products) {
  const counted = Number(counts[product.id])
  const system = snapshot[product.id]
  if (isNaN(counted) || counted === system) continue
  const delta = counted - system
  await registerMovement({
    product_id: product.id,
    action: delta > 0 ? 'in' : 'loss',
    quantity: Math.abs(delta),
    notes: `Ajuste de inventário - Contagem: ${counted}, Sistema: ${system}`
  })
}
```

### UX de navegação por teclado
```tsx
<input
  type="number"
  min={0}
  tabIndex={index + 1}
  value={counts[product.id] ?? ''}
  onChange={e => setCounts(prev => ({ ...prev, [product.id]: e.target.value }))}
  className="w-24 text-center"
/>
```

### Nota sobre snapshot
O snapshot é apenas um objeto em memória durante a sessão. Não é persistido no banco. Se a página for atualizada durante o inventário, o processo recomeça. Para MVP isso é aceitável.
