## Story
**As a** gestor do Mr. Lion,
**I want** filtrar o histórico de movimentações e relatórios por usuário/operador,
**so that** eu acompanhe a produtividade e rastreie quem realizou cada operação para fins de auditoria.

## Acceptance Criteria

1. A página de Relatórios tem um filtro "Operador" com dropdown listando todos os usuários que realizaram movimentações
2. Ao selecionar um operador, a tabela exibe somente movimentações realizadas por aquele usuário
3. O filtro de operador se combina com os demais filtros (produto, tipo, data)
4. A coluna "Usuário" em Relatórios exibe o nome completo (não o ID)
5. Em mock mode, os dados mock incluem pelo menos 2 usuários distintos nas movimentações
6. O filtro de operador está disponível apenas para managers (operators não veem este filtro)
7. O dashboard exibe quem foi o operador mais ativo do dia (last 24h)
8. O export CSV inclui a coluna de usuário com nome completo
