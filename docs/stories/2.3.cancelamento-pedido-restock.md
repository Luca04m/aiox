## Story
**As a** gestor do Mr. Lion,
**I want** cancelar um pedido e ter o estoque restaurado automaticamente,
**so that** erros de cadastro ou pedidos cancelados pelo cliente não gerem divergências no estoque.

## Acceptance Criteria

1. Pedidos com status `pending` ou `confirmed` têm botão "Cancelar Pedido" visível para managers
2. Ao clicar em cancelar, um dialog de confirmação aparece listando os itens que serão devolvidos ao estoque
3. O dialog tem dois botões: "Cancelar Pedido" (confirma, vermelho) e "Manter Pedido" (fecha)
4. Ao confirmar cancelamento, o status do pedido muda para `cancelled`
5. Para cada item do pedido cancelado, uma movimentação de `in` é criada com nota: `"Devolução por cancelamento do pedido #MRL-XXXX"`
6. O estoque de cada produto é restaurado automaticamente (restock)
7. Pedidos `delivered` NÃO podem ser cancelados (botão não aparece)
8. Pedidos `cancelled` aparecem em uma aba separada "Cancelados" na OrdersPage
9. O cancelamento é atômico: ou tudo é revertido, ou nada muda
10. Toast de sucesso indica quais produtos tiveram estoque restaurado

### Mutation useCancelOrder
```typescript
export function useCancelOrder() {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: async ({ orderId, order }: { orderId: string, order: DeliveryOrder }) => {
      if (IS_MOCK) {
        // atualizar mockOrders e criar mockMovements de restock
        const orderIndex = _mockOrders.findIndex(o => o.id === orderId)
        if (orderIndex !== -1) _mockOrders[orderIndex].status = 'cancelled'
        for (const item of order.items) {
          _mockMovements.unshift({
            id: nanoid(),
            product_id: item.product_id,
            action: 'in',
            quantity: item.quantity,
            notes: `Devolução por cancelamento do pedido ${formatOrderNumber(orderId)}`,
            user_id: 'mock-user',
            order_id: orderId,
            created_at: new Date().toISOString()
          })
        }
        return
      }
      // Supabase: atualizar status + criar movimentações de restock
    },
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['delivery_orders'] })
      queryClient.invalidateQueries({ queryKey: ['products'] })
      queryClient.invalidateQueries({ queryKey: ['stock_movements'] })
      toast.success('Pedido cancelado. Estoque restaurado.')
    }
  })
}
```
