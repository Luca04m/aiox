## Story
**As a** gestor do Mr. Lion,
**I want** uma tela dedicada para visualizar e analisar perdas de estoque,
**so that** eu identifique padrões, produtos mais afetados e tome decisões para reduzir desperdícios.

## Acceptance Criteria

1. A página de Relatórios tem uma tab "Perdas" separada das movimentações gerais
2. A tab Perdas exibe somente movimentações do tipo `loss`
3. Um painel de KPIs mostra: total de perdas no período, número de produtos afetados, produto com mais perdas
4. Um gráfico de barras exibe os top 5 produtos com maior quantidade perdida no período
5. A tabela de perdas tem colunas: Data, Produto, SKU, Quantidade Perdida, Responsável, Motivo (notas)
6. Filtros de data (de / até) se aplicam à tab Perdas
7. O período padrão é os últimos 30 dias
8. Export CSV dos dados de perdas do período filtrado
9. Destacar visualmente produtos com perdas recorrentes (mais de 3 registros no período)
10. Exibir "R$ estimado" por linha se produto tiver custo cadastrado (campo futuro — deixar placeholder)

### Calcular produto mais afetado
```typescript
function getMostLostProduct(movements: StockMovement[]): { name: string, quantity: number } | null {
  if (!movements.length) return null
  const byProduct: Record<string, { name: string, quantity: number }> = {}
  for (const m of movements) {
    const name = m.product?.name ?? m.product_id
    byProduct[m.product_id] = {
      name,
      quantity: (byProduct[m.product_id]?.quantity ?? 0) + m.quantity
    }
  }
  return Object.values(byProduct).sort((a, b) => b.quantity - a.quantity)[0]
}
```

### Destaque de produto recorrente
```typescript
const lossCountByProduct: Record<string, number> = {}
for (const m of lossMovements) {
  lossCountByProduct[m.product_id] = (lossCountByProduct[m.product_id] ?? 0) + 1
}
const isRecurrent = (productId: string) => (lossCountByProduct[productId] ?? 0) > 3
```
