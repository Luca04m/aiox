## Story
**As a** operador do Mr. Lion,
**I want** registrar entrada de múltiplos produtos em uma única operação,
**so that** eu não precise repetir o fluxo de entrada produto por produto ao receber uma entrega com vários itens.

## Acceptance Criteria

1. A página `/entrada` oferece dois modos: "Entrada Simples" (atual) e "Entrada em Lote"
2. No modo lote, o usuário pode adicionar múltiplas linhas produto + quantidade
3. Cada linha exibe: select de produto, campo de quantidade (stepper), campo de origem, e botão remover linha
4. O usuário pode adicionar quantas linhas quiser com botão "+ Adicionar Produto"
5. Não é possível selecionar o mesmo produto duas vezes na mesma lote (validação)
6. A tela de confirmação lista todos os itens do lote com estoque atual → novo estoque para cada um
7. Ao confirmar, todas as movimentações são registradas atomicamente (se uma falhar, nenhuma é salva)
8. Toast de sucesso indica "X entradas registradas com sucesso"
9. Após confirmação bem-sucedida, o formulário é resetado para um novo lote
10. Modo offline: entradas em lote são salvas no Dexie e sincronizadas ao reconectar

Se `Promise.all` falhar parcialmente, logar erro e exibir quais itens falharam. Rollback completo pode ser complexo — para MVP, mostrar quais falharam e deixar usuário corrigir manualmente. Log de erro detalhado no toast.
