## Story
**As a** gestor do Mr. Lion,
**I want** ver um resumo estatístico no topo da página de relatórios mostrando totais por tipo de movimentação e por categoria,
**so that** eu tenha uma visão rápida dos números do período sem precisar contar manualmente as linhas da tabela.

## Acceptance Criteria

1. Um painel de resumo aparece acima da tabela de histórico com totais calculados dos dados filtrados
2. O resumo exibe contagem e quantidade total para cada tipo: Entradas, Saídas, Ajustes, Perdas
3. Cada tipo tem cor correspondente: Entradas (verde), Saídas (vermelho), Ajustes (âmbar), Perdas (laranja)
4. O resumo exibe distribuição por categoria (honey, cappuccino, blended, acessorio) com totais de movimentações
5. O resumo atualiza dinamicamente conforme os filtros são alterados
6. O total de itens encontrados é exibido claramente ("42 movimentações encontradas")
7. Em mobile, o painel de resumo usa layout grid 2×2 para os tipos e lista vertical para categorias
8. Os totais de quantidade somam as quantidades (não apenas conta os registros)

### Tipo MovementSummary
```typescript
interface TypeStat {
  count: number
  quantity: number
}
interface MovementSummary {
  in: TypeStat
  out: TypeStat
  adjustment: TypeStat
  loss: TypeStat
  byCategory: Record<string, number>
  total: number
}
```

### Função calculateMovementSummary
```typescript
export function calculateMovementSummary(movements: StockMovement[]): MovementSummary {
  const summary: MovementSummary = {
    in: { count: 0, quantity: 0 },
    out: { count: 0, quantity: 0 },
    adjustment: { count: 0, quantity: 0 },
    loss: { count: 0, quantity: 0 },
    byCategory: {},
    total: movements.length
  }
  for (const m of movements) {
    summary[m.action].count++
    summary[m.action].quantity += m.quantity
    if (m.product?.category) {
      summary.byCategory[m.product.category] = (summary.byCategory[m.product.category] || 0) + 1
    }
  }
  return summary
}
```
